# Structure & editorial rules

Form follows argument. Decide what each slide *says* before how it looks.

## Assertion titles (the single biggest lever)
Every content slide's title is a short, complete statement of the takeaway —
not a topic noun.

| Topic label (weak) | Assertion title (strong) |
|---|---|
| References clients | Des leaders banque & assurance nous renouvellent leur confiance |
| Power BI | Leader du Gartner Magic Quadrant 2015–2024, notre standard |
| Méthodologie | La donnée fiable d'abord, la visualisation ensuite |

Ghost-deck test: read only the titles, in order. They must tell the whole story.
If a title could sit on any slide, it's a label — rewrite it.

## One idea per slide
Each slide does exactly one job. If it does two, split it. If a slide has no
single takeaway, cut it. Three to five supporting points max.

## One fact, one home (de-duplication)
Repetition is the enemy of a tight deck. Before writing, list every recurring
proof point and assign each to ONE slide; delete it elsewhere.

Worked example (consulting plaquette):
| Proof | Single home | Removed from |
|---|---|---|
| Founded 2018 | "The firm" intro slide | cover, references, trust slide |
| 20+ clients (headline number) | cover hero | intro, trust slide |
| 5/5 client rating | trust slide (tied to the rating system) | intro |
| Certification / award | the dedicated engagement/expertise slide | cover, intro |

The cover may *assert* the headline number; the trust slide *proves* it with
logos — that's two roles for one fact, which is fine. Restating the same
sentence three times is not.

## Recover, don't reinvent
If an approved source deck exists, treat its content as the source of truth.
Keep the substance; restructure and modernise the form. Mine it for material the
current draft dropped (values, proof points, references) rather than writing new
claims. Never invent figures about a real organisation — ask or pull from source.

## Deck architecture (typical business/marketing deck)
1. Cover — one promise + one headline number.
2. The firm / who we are — positioning, the one founding fact, what makes us different.
3. Values / principles (optional) — detailed vignettes, filled.
4. Differentiator — the single sharpest reason to choose us.
5. Offering / practices — overview, then one slide per practice (vary the layout).
6. Method / how we work.
7. Proof — references (group by theme; show logos; de-dup repeated clients).
8. Trust — logo wall + a testimonial + the headline metric's single home.
9. Engagement / commitments — certifications, CSR (their single home).
10. Contact / CTA.

## Vary the layout across archetypes
Never the same card grid on every slide. Rotate:
- numbered 2×2 cards (pillars/steps)
- split: coloured panel left / structured list right (a practice in depth)
- horizontal flow with an arrow (a 2-step method)
- big-number / stat hero (a breather)
- logo wall / matrix (references, trust)
- full-bleed quote (a breather)

## QA checklist (run every time)
- [ ] Every content slide has an assertion title; ghost-deck test passes.
- [ ] No fact repeated across slides (one-fact-one-home respected).
- [ ] No card/box with an idle void (fill rule respected).
- [ ] Body text ≥ 14pt; nothing unreadable.
- [ ] One palette, one style recipe; outer radius ≥ inner radius.
- [ ] No stray running footers (unless asked).
- [ ] Layouts vary across archetypes; the deck doesn't look monotonous.
- [ ] Rendered every changed slide and eyeballed it.
